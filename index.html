What it sees
01
Tasks & Subtasks
Sticky notes, bullet points, circled items → structured work packages with titles and descriptions
02
Dependencies & Arrows
Lines, arrows, "after X" annotations → predecessor/successor relationships in the Gantt
03
People & Assignments
Names, initials, team labels → assigned owners on each task
04
Dates & Milestones
\"Week 3\", \"March\", \"MVP\" → start/end dates, milestone markers, sprint boundaries
05
Priority & Urgency
Stars, exclamation marks, red markers, underlines → priority levels on work packages

openclaw --explain architecture
Under the hood
01
Vision Analysis
Claude's vision model reads the whiteboard image — handwriting, diagrams, spatial layout, colors, arrows. Everything.
02
Context Engineering
OpenClaw skills provide the project management context: your team roster, existing project structure, naming conventions, tool constraints.
03
Structured Extraction
Tasks, dependencies, milestones, owners extracted into a clean JSON structure. Human reviews before push.
04
API Push
One command pushes everything to OpenProject/Jira: creates tasks, sets relations, assigns people, marks milestones.
